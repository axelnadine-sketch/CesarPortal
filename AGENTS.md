# AGENTS.md

## Project rules
- Priorité à la simplicité, à la maintenabilité et à la clarté.
- Ne pas ajouter de dépendances inutiles.
- Préférer une V1 réellement exploitable.
- Toujours protéger les routes admin.
- Toujours valider les données côté serveur.
- Toujours garder le code lisible et bien structuré.
- Préférer imageUrl à un système d’upload complexe pour la V1.
- Avant les gros changements, vérifier l’état Git.
- Après chaque phase importante, lancer les vérifications utiles si disponibles.
- Ne pas casser l’architecture pour gagner 2 minutes.
